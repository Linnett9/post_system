# Post System

Generates daily Facebook-style food posts for Deelicious Eats using OpenRouter, then sends them to Telegram.

## Quick Start

From the project folder:

```bash
cd /Users/brandonlinnett/Desktop/post_system
python3 main.py
```

`main.py` generates 3 ads, sends them to Telegram, then exits.

## Setup

Install the Python packages this project uses:

```bash
pip3 install requests python-dotenv pytest
```

Create a `.env` file in the project root:

```env
OPENROUTER_API_KEY=your_openrouter_api_key
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
TELEGRAM_CHAT_ID=your_telegram_chat_id
```

The `.env` file is loaded by `main.py`.

## Main Files

- `main.py` - generates 3 ads, sends them to Telegram, then exits
- `.github/workflows/daily_ads.yml` - runs the bot once a day using GitHub Actions
- `application/post_orchestrator.py` - selects intent, menu item, offer, builds the prompt, and returns generated posts
- `domain/interfaces/prompt_builder.py` - controls the LLM instructions and post style
- `infrastructure/llm/openrouter_client.py` - sends prompts to OpenRouter
- `infrastructure/messaging/telegram_notifier.py` - sends posts to Telegram
- `menu.json` - menu items used for post generation
- `item_history.json` - tracks recently used menu items

## GitHub Actions Daily Run

This repo includes:

```text
.github/workflows/daily_ads.yml
```

The workflow runs:

```bash
python main.py
```

It is scheduled with:

```yaml
cron: "0 7 * * *"
```

GitHub Actions cron uses UTC. During British Summer Time, `7:00 UTC` is `8:00 AM` in the UK. During winter, change it to:

```yaml
cron: "0 8 * * *"
```

You can also run it manually in GitHub:

```text
Actions -> Daily Ads -> Run workflow
```

Add these repository secrets in GitHub:

```text
Settings -> Secrets and variables -> Actions -> New repository secret
```

Required secrets:

```text
OPENROUTER_API_KEY
TELEGRAM_BOT_TOKEN
TELEGRAM_CHAT_ID
```

## Removing `****` Stars From Posts

The stars come from Markdown bold formatting generated by the LLM.

This is handled in two places:

- `domain/interfaces/prompt_builder.py` tells the LLM not to use Markdown, asterisks, headings, bullets, or numbered lists.
- `application/post_orchestrator.py` cleans generated content before it is printed or sent:

```python
content.replace("****", "").replace("**", "").strip()
```

If the model still adds bold markers, the cleanup step removes them automatically.
